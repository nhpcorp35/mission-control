
# Mission Control

Mission Control is a private AI development workflow system.

It converts engineering objectives into safe, reproducible missions for coding agents while preserving repository isolation, review, and human approval.

Its immediate purpose is to reduce friction in the development of Legal AI.

Its long-term vision is to become a general AI project operating system, proven first through the development of Legal AI.

## Priorities

1. Legal AI remains the primary project.
2. Mission Control must save Legal AI development time.
3. Mission Control must not become a distraction from attorney validation, product usefulness, or revenue.
4. Features are added only when they solve a demonstrated workflow problem.

## Roles

- Allen sets priorities and makes final decisions.
- Hal acts as Technical Lead.
- Mission Control enforces the approved mission.
- Coding agents perform authorized repository work.
- No coding agent may commit or push without explicit approval.

## Current Phase

Architecture and mission specification.

No executable implementation has begun.
